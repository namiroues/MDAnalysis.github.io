MDAnalysis is an open-source project that welcomes contributions from the community. Whether you're a developer, a documentation writer, or someone interested in improving the website, there are many ways to get involved.

You can contribute by **developing new features, fixing bugs, improving documentation, enhancing tutorials, or engaging in discussions**. If you're new, you can start by participating in community conversations and mentoring programs. For more information see [Community]({{ site.baseurl }}/pages/community/) and [Events]({{ site.baseurl }}/pages/events/).

This page provides an overview of how you can contribute to the [Code](#contributing-to-the-code), the [Documentation](#contributing-to-documentation), or the [Website](#contributing-to-the-website).

MDAnalysis maintains **three GitHub repositories**, each dedicated to a specific area of contribution:

- [Main Code Repository][]: Used for code development and API documentation.
- [User Guide Repository][]: Contains installation instructions, tutorials and learning resources.
- [Website Repository][]: Hosts the website and blog.

## Contributing to the Code

If you're interested in developing new features, fixing bugs, or improving performance, contributing to the MDAnalysis codebase is a great way to get involved.

To get started:
- Read the [Contributing to the Main Codebase][] guide, which outlines all the necessary steps.
- Browse [GitHub Issues][] to find open tasks (look for "good first issue" if you're a beginner).
- If you find a bug or want to suggest improvements, please report it via the [Issue Tracker]({{ site.github.issues }}).
- Follow best practices, including [writing and running tests][] and [update the documentation][] for any changes.

## Contributing to Documentation

Good documentation is essential to making MDAnalysis accessible to users and developers. MDAnalysis maintains two kinds of documentation the [User Guide][] and the [API Reference][]. 

### Improving the User Guide

The [User Guide][] provides tutorials and explanatory content to help users install and learn MDAnalysis. You can contribute updating unclear explanations or adding new tutorials or examples. 

To contribute to the User Guide:
- Follow the [Contributing to the User Guide][].
- Check for open documentation-related issues in the [User Guide GitHub repository][].

### Improving API Documentation

API documentation is automatically generated from *docstrings* in the codebase and provides reference material for developers. You can contribute by improving *docstrings* for functions, classes, and modules or updating outdated documentation.

To contribute to the API documentation:
- Follow the [API Documentation Contribution Guide][].
- Browse open API doc issues in [GitHub Issues][].

To get involved you can go through the [contributing to the user guide][] instructions. 

## Contributing to the Website

MDAnalysis maintains its own website and blog, which are also open for contributions. If you're interested in improving content, fixing typos, or adding new pages, you can help enhance the website.

To contribute to the website see the [README file](https://github.com/MDAnalysis/MDAnalysis.github.io) in the [Website Repository][] for instructions.

If you’re interested in writing a **blog post**, reach out to the MDAnalysis team on [GitHub Discussions][] or [Discord][].
